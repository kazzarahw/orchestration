# Use Todo

<IRON-LAW>
For any task with ≥3 steps, the todo list is your SOURCE OF TRUTH for progress — not memory.
Create it before starting; each lifecycle phase, gate, and skill-checklist item becomes a todo.
</IRON-LAW>

## Why it is load-bearing
- **Durable state:** after context compaction, trust the todo list (with `git log` and the SDD
  progress ledger) over recollection — this is what keeps steps from being skipped or repeated.
- **Commitment:** an open, visible todo is what makes a step impossible to silently drop.
  Announcing a skill commits you verbally; the todo commits you structurally.

## How to use
- One item `in_progress` at a time.
- Mark complete only after verification — evidence, not "should be done".
- Blocked? Keep it `in_progress` with a follow-up naming the blocker.
- Specific, actionable items; break large work down.
- Update it as you go — a stale list is worse than none.
